# Ленивая загрузка тяжёлых зависимостей

Этот документ посвящён ленивой загрузке **крупных библиотек и модулей**, которые используются только в отдельных частях приложения: графики, rich‑text‑редакторы, карты, сложные UI‑наборы.

## Проблема

- Тяжёлые библиотеки существенно увеличивают размер бандла.
- Если такая библиотека нужна только на нескольких страницах, включение её в стартовый бандл необоснованно.
- Даже при ленивой загрузке компонента важно убедиться, что тяжёлая зависимость не «просочилась» в основной чанк.

## Базовый приём: изоляция тяжёлой зависимости

Классический подход:

1. Вынести использование библиотеки в отдельный компонент/модуль.
2. Загрузить этот компонент лениво через `React.lazy` и `import()`.

Пример:

```tsx
// HeavyChart.tsx
import React from 'react';
import { Chart } from 'some-heavy-chart-lib';

export default function HeavyChart() {
  return <Chart data={/* ... */} />;
}
```

```tsx
// ленивый импорт HeavyChart
const HeavyChart = React.lazy(() => import('./HeavyChart'));

function ReportsPage() {
  return (
    <React.Suspense fallback={<div>Загрузка графика...</div>}>
      <HeavyChart />
    </React.Suspense>
  );
}
```

Бандлер вынесет:

- код `HeavyChart` и
- код `some-heavy-chart-lib`

в отдельный чанк (или несколько чанков, в зависимости от настроек).

## Управление повторным использованием библиотек

Если тяжёлая библиотека используется в нескольких модулях:

- бандлер может создать общий чанк с библиотекой и несколько зависимых чанков для конкретных компонентов;
- это уменьшает дублирование кода, но добавляет сетевые запросы.

Практические рекомендации:

- сгруппировать использование тяжёлой библиотеки в ограниченном числе модулей;
- избегать импортов тяжёлой библиотеки:
  - в стартовом коде приложения,
  - в часто используемых лёгких компонентах.

## Интеграция с менеджером компонентов

Вместе с `ComponentLoader` можно:

- описать отдельные записи для тяжёлых виджетов;
- централизованно контролировать:
  - когда их подгружать,
  - какие fallback‑компоненты показывать,
  - какие метрики отправлять.

Это удобно, если:

- тяжёлых виджетов много;
- важно отслеживать, насколько часто они используются и как долго загружаются.
